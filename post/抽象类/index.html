抽象类

为什么需要抽象类

假如我们有一个父类：

public class Animal{
    public void Run(){
        System.out.println("我不知道怎么跑")//每个动物跑法不一样
    }
}

还有两个子类:

public class Dog(){
    public void Run(){
        System.out.println("我拿四条腿跑步")//狗是四脚
    }
}
public class Duck(){
    public void Run(){
        System.out.println("我两条腿跑")//鸭子两只脚
    }
}

这样我们便会发现，父类的Run方法就没有什么用处，只是给Duck和dog两个类去继承和重写该方法。

这样的话，为什么不拥有一个机制？让它能够应对父类方法的不确定性。

于是乎，Java里面便提供了一个关键字，abstract。

通过用abstract关键字来修饰该类，这个类就是抽象类。

抽象方法

就像上面那个Run方法，在父类中它并没有什么用处，它只会给子类提供一个名字和返回值让子类去重写该方法。那么我们可以将该方法设置成为一个抽象方法.

抽象方法的声明:


abstract 返回值 名字();

在返回值前面加上abstract关键字,便将这个方法设置成为一个抽象方法。

抽象类

如果一个类里面有一个或多个抽象方法，那么这个类就是必须设置成为一个抽象类，这样就要在class前面加上abstract关键字

public abstract class 类名{

}

抽象类不能够被实例化，如果要实例化，就要靠子类继承抽象类，并重写父类所有的抽象方法。

如果不这么做，那么这个类也必须标记为抽象类，也要加上abstract关键字，那么这个子类也是无法被实例化的，因为它还是一个抽象类。

当然，我们也可以创建了一个没有任何抽象方法的抽象类。比如有些类之间创建对象是没有什么意义的，我们可以通过此方法阻止它产生它的对象。

来个例子感受一下

abstract class Instrument {//写一个乐器抽象类
     private int i;

     public abstract void play();//两个抽象方法
     public abstract void adjust();

     public String what(){return "Instrument";}
}

class Wind extends Instrument{//这里管乐器继承乐器类

    @Override
    public void play() {//普通类继承抽象类，就实现这些抽象方法
        System.out.println("Wind play");
    }

    @Override
    public void adjust() { }//这里什么都不写

    public String what(){//重写what函数
        return "Wind";
    }
}

class Percussion extends Instrument{

    @Override
    public void play() {//普通类继承抽象类，就实现这些抽象方法
        System.out.println("Percussion play");
    }

    @Override
    public void adjust() { }

    @Override
    public String what() {//重写what函数
        return "Percussion";
    }
}

class WoodWind extends Wind{
    @Override
    public String what() {
        return "WoodWind";
    }

    @Override
    public void play() {
        System.out.println("WoodWind play");
    }
}

测试类:

public class Main {

    public static void main(String[] args) {
      Instrument wind=new Wind();
      Instrument WoodWind=new WoodWind();
      Instrument Percussion=new Percussion();
      wind.play();
      WoodWind.play();
      Percussion.play();
    }
}

对抽象的理解

引用知乎里对抽象类的一个说法

至于抽象类，它和抽象画还真的有点相似之处

抽象画：我告诉你我画了一幅画，具体画的啥，我也不知道

抽象类：我告诉你我定义了一些方法，方法具体做什么，我也不知道！
