SQL语句(一)

Jan 8, 2019 6 min read java, 数据库

SQL语句(一)

sql语言包含:

数据定义语言(DDL),它提供定义关系模式、删除关系以及修改关系模式的命令

数据操纵语言(DML),它包括查询语言，以及往数据库中插入元组、从数据库中删除元组和修改数据库中元组的命令

创建数据库

用语句创建数据库

CREATE DATABASE database_name
ON
{[PRIMARY｝
(
    NAME = logical_file_name,
    FILENAME = 'os_file_name'
    [,SIZE = size]
    [,MAXSIZE = max_size]
    [,FILEGROWTH = growth_increment,]
)
[,..n]


LOG ON
(
    NAME = logical_name,
    FILENAME = 'os_file_name'
    [SIZE = size]
        [,MAXSIZE = max_size]
    [,FILEGROWTH = growth_increment,]
)
[,..n]

database_name ：表示为数据库名字

PRIMARY: 用于指定主文件组的文件，一个数据库只能有一个主文件

NAME ：指定数据库的逻辑名称

FILENAME：指定数据库所在文件的操作系统文件名称和路径

SIZE：指定数据库的初始容量大小

MAXSIZE： 指定操作系统文件可以增长到的最大尺寸

FILEGROWTH: 指定文件增量大小

修改数据库

ALTER DATABASE database_name
{ADD FILE <filespec>[,…n] [TO FILEGROUP filegroup_name]
ADD LOG FILE  <filespec>[,…n]
REMOVE FILE logical_file_name
REMOVE FILEGROUP filegroup_name
MODIFY FILE <filespec>
MODIFY NAME=new_database_name
ADD FILEGROUP filegroup_name
MODIFY FILEGROUP filegroup_name
<filespec>::=
（|NAME=logical_file_name
[,NEWNAME=new_logical_file_name]
[,FILENAME=’os_file_name’]
[,SIZE=size]
[,MAXSIZE={max_size|UNLIMITED}]
[,FILEGROWTH=growth_increment,])

ADD FILE <filespec>[,…n] [TO FILEGROUP filegroup_name]： 表示向指定的文件组中添加新的数据文件。

ADD LOG FILE <filespec>[,…n]：增加新的日志文件。

REMOVE FILE logical_file_name：删除指定的操作系统文件。

REMOVE FILEGROUP filegroup_name：删除指定的文件组。

MODIFY FILE <filespec>：修改某个操作系统文件。

MODIFY NAME=new_databasename：重命名数据库。

ADD FILEGROUP filegroup_name：增加一个文件组。

MODIFY FILEGROUP filegroup_name：修改某个指定文件组的属性。

删除数据库

DROP DATABASE database_name

创建表

使用CREATE TABLE语句创建表

创建一个department表
    CRAETE TABLE department
    (
        dept_name varchar(20),
        building varchar(15),
        budget numeric(12,2),
        primary key(dept_name);
    )

database_name－ 是要在其中创建表的数据库名称。

Owner－ 是新表所有者的用户 ID 名,由这些用户创建的表的默认所有者为 dbo

table_name－ 是新表的名称。表名必须符合标识符规则。数据库中的 owner.table_name 组合必须唯一。table_name 最多可包含 128 个字符

column_name－是表中的列名。列名必须符合标识符规则，并且在表内唯一。

列的属性－包括列的数据类型、列的长度、列上的约束（CONSTRAINT 是可选关键字，表示 PRIMARY KEY、NOT NULL、UNIQUE、FOREIGN KEY 或 CHECK 约束定义的开始。约束是特殊属性，用于强制数据完整性并可以为表及其列创建索引。）

primary key(A1,A2,…) 表示属性A1，A2构成主码，关系中没有一元组在主码上取值相同。

foreign key(A1,A2) references: 属性A1，A2上的取值必须对应于关系S中某元组在主码属性上的取值

not null : 一个属性上的not null约束表明在该属性上不允许空值

插入数据

insert into table values (co1_value,co2_value,co3_value)

元组的属性与关系模式中属性排列顺序一致。如果忘记,可以在insert语句中指定属性

insert into table(co1,co2,co3)
values (co1_value,co2_value,co3_value)

更新

update r
set q
where p

update 语句可以包含select语句在where字句中的任何合法结构

删除

删除表是 drop table

删除数据是:

delete from r where P

查询

单关系查询

select Co_name1,Co_name2,...
from table_name
where condition

Co_name 是该表的一个属性 table_name是名字 condition 是条件

多关系

select tab1_Co_name1,tab2_co_name2
from tab1,tab2
where tab1.co_name1=tab2.co_name1

select 用于列出查询结果中所需要的属性

from是一个查询求值中需要访问的关系列表

where是一个作用在from 子句中关系的属性上的谓词

自然连接

自然连接作用域两个关系，并产生一个关系作为结果

比如
select name,course_id
from instructor,teaches
where instructor.ID = teaches.ID

用自然连接运算可写成

select name,course_id
from instructor natural join teaches

两个查询结果相同

一个from字句中，可以用自然连接将多个关系结合在一起

    select A1,A2,..,An
    from r1 natural join r2 natural join r3 .... natural join rm
    where p;

附加的基本运算

更名运算

old-name as new-name as字句既可以出现在select中，也可以出现在from字句中

字符串运算

百分号(%):匹配任意子串

下划线(_):匹配任意一个字符

’%Comp’ 匹配任何包含Comp子串的字符串

‘_ _ _‘匹配只含3个字符的字符串

‘_ _ _%‘匹配至少含3个字符的字符串

排列元组的显示次序

order by 字句让查询结果中的元组按排列顺序显示

order by字句默认使用升序，可以用desc表示降序，或用asc表示升序。

例：

按salary的降序列出整个instructor，如果几位老师工资相同，按姓名升序排列
select *
from instructor
order by salary desc, name asc;

集合运算

并运算

为了找出在2009年秋季开课，或者在2010年春季开课或两个学期都开课的所有课程

(
    select course_id
    from section
    where semester = 'Fall' and year = 2009
 )
union
(

交运算

找出在2009年秋季和2010年春季同时开课的所有课程的集合

(
    select course_id
    from section
    where semester = 'Fall' and year = 2009
)
intersect
(
    select course_id
    from section
    where semester = 'spring' and year =2010

差运算

)
except

    from section

)

聚集函数

聚集函数是以值的一个集合为输入返回单个值的函数。提供了五个固有聚集函数：

  • 平局值：avg
  • 最小值：min
  • 最大值：max
  • 总和：sum
  • 计数：count

基本聚集

select avg(salary)
from instructor
where dept_name = 'Comp.Sci';

查询结果是一个具有单属性的关系，只包含一个元组。这个元组对应教师平均工资

分组聚集

有时候我们不仅希望聚集函数作业在单个元组集上，也希望将其作用到一组元组集上，可用group by字句实现。

group by 字句给出一个或多个属性用来构造分组的，group by子句中的所有属性上的取值相同的元组将被分在一个组中

找出每一个系的平均工资
select dept_name,avg(salary)
from instructor
group by dept_name;

该元组按照dept_name属性进行分组的。

当SQL查询使用分组时，一个很重要的事是保证出现在select语句中但没有被聚集的属性只能是出现在group by字句中的那些属性.

having 字句

教师平均工资超过42000美元
select dept_name,avg(salary)
from instructor
group by dept_name
having avg(salary)> 42000

having 字句是对分组的限定条件

嵌套子查询

子查询是另一个查询中的select-from-where表达式

集合成员资格

连接词 in 测试元组是否是集合中的成员。 not in 测试元组是否不是集合中的成员

查找2009年秋季开课的课程与2010年春季开课的课程集合

select course_id
from section
where semester = 'Fall' and year = 2009 and
course_id in(select course_id
    from section
    where semester = 'spring' and year = 2010
)

集合的比较

select name
from instructor
where salary > some (
    select salary
    from instructor
    where dept_name = 'Biology'
)
当元组的salary值比Biology系教师的所有工资值集合中的某一成员高，外层select 的where字句中>some 的比较为真

sql 也运行 <some,<=some,>=some,=some和<>some的比较

结构>all 对应于词组“比所有的都大”

空关系测试

exists 结构在作为参数的子查询非空时返回true值。

来自外层查询的一个相关名称可以用在where字句的子查询中，使用了来自外层查询相关名称的子查询被称作相关子查询。

我们可以用not exists 测试子查询结果集中是否不存在元组。

from字句中的子查询

sql运行from字句中使用子查询表达式。

select dept_name,avg_salary
from (select dept_name,avg(salary)as avg_salary
    from instructor
    group by dept_name
    )
where avg_salary > 42000;
