单片机的某些概念

  • 位数： 单片机能够一次处理的数据的宽度
  • 存储器：包括 程序存储器 和 数据存储器 。 程序存储器空间较大，字节数一般从几kb到几十kb。
  • 工作电压：通常工作电压是5v
  • 功耗：μA或nA

单片机内部结构

中央处理器（CPU）

运算器

主要包括算术逻辑运算部件（ALU）、累加器A、寄存器B、程序状态字（PSW）、暂存器、布尔处理器以及十进制调整电路等。

算术逻辑部件（ALU）

主要对数据进行算术运算的：

  • 加、减、乘、除运算
  • 增量（加1）、减量（减1）运算
  • 十进制数调整
  • 位操作中的置位、复位和取反操作
  • 与、或、异或等运算操作
  • 数据传送操作

累加器A

累加器A是cpu中最繁忙、使用频率最高的一个特殊功能寄存器

其作用为：

  • 累加器A作为ALU的输入数据源之一，也是ALU的输出。
  • CPU中的数据传送大多数都通过累加器，累加器A是一个非常重要的数据中转站。

寄存器B

只有用乘或除才用

执行乘法时，用于存放其中一个乘数和乘积的高8位数 执行除法时，用于存放除数和余数

程序状态字（PSW）

| D7 | D6 | D5 | D4 | D3 | D2 | D1 | D0 | | - | - | - | - | - | - | - | - | | CY | AC | F0 | RS1 | RS0 | OV | - | P |

CY: 进位标志

进行算术运算时，由硬件置位或复位,表示运算过程中，最高位是否有进位或借位状态

进行位操作时，CY被认为是位累加器，它的作业相当于CPU中的累加器A

AC: 辅助进位标志

进行加法或减法运算时，若低4位向高4位有进位或借位，AC将被硬件置1，否则置0.

AC位常用于十进制调整指令和压缩BCD运算等

F0： 用户标志位

作为一个用户自定义的状态标志

RS1、RS0： 工作寄存器组选择位

| RS1、RS0 | 寄存器组（地址单位） | | - | -| | 00 | 寄存器组0（00H-07H） 1区 | | 01 | 寄存器组1（08H-0FH） 2区 | | 02 | 寄存器组2 （10H-17H） 3区 | | 03 | 寄存器组3（18H-1FH） 4区 |

OV： 溢出标志位

算术运算时：产生溢出，将OV置1，溢出为真，表明结果超出寄存器A表示的符号数范围，否则清0，溢出为假

加减运算时：采用双进位的状态来判断，双进位标志是指Cp和Cs。若Cp异或Cs=0，表示无溢出，0v=0.反正，ov=1

无符号溢出看CY，有符号数溢出看OV

P： 奇偶标志位 >该位用来表示累加器A中为1的位数是奇数还是偶数。 > >若累加器A中为1的位数是奇数，则P标志位置1，否则P标志位清0.

控制器

程序计数器(PC)

程序计数器是一个独立的计数寄存器，存放下一条要从程序存储器中取出指令的地址。

51系列单片机的程序计数器为16位。

程序计数器的工作不完全是顺序的，因为在指令中，存在转移、子程序调用、中断调用返回等工作，程序计数器就不再是自动加“1”了。

数据指针(DPTR)

数据指针是一个16位专用寄存器

数据指针寄存器也可用做访问程序存储器时的基址寄存器，还可以作为一个通用的16位寄存器或两个8位寄存器使用。

指令寄存器(IR)、指令译码器(ID)以及控制逻辑

指令寄存器是用来存放操作码的专用寄存器。指令译码器译码识别IR中指令的操作类型。控制逻辑从取指令开始，直至指令执行控制各部件协调工作。

存储器结构

程序存储器：片内程序存储器和片外程序存储器

数据存储器：片内数据存储器和片外数据存储器

内部程序存储器

如果EA引脚接地，程序读取只从外部存储器开始。

如果EA接至VCC，程序读写先从内部存储器（地址为0000H~1FFFH）开始，接着从外部寻址，寻址地址为2000H~FFFFH。

内部数据存储器

通用寄存器区（00H~1FH）

每块包含8个8位寄存器，均以R0~R7来命名，我们常称这些寄存器为通用寄存器。

位寻址区（20H~2FH）

即可作为一般单元用字节寻址，也可以对它们的位进行寻址。 （可拿8位，可拿1位）

CPU能直接寻址这些位，执行例如置“1”、清“0”、求“反”、转移、传送和逻辑等操作。

数据缓存器区（30H~7FH）

用于0存放用户数据和各种字节标志，以采用直接或间接寻址方式访问。

堆栈指针

51系列单片机属于向上生长型堆栈，这种堆栈操作规则如下：

  • 进栈操作：先SP加1，后写入数据。
  • 出栈操作：先读出数据，后SP减1。

I/O端口结构

51系列单片机有1个8位双向并行I/O端口P0和3个8位准双向并行I/O端口P1~P3.

P0端口

P0口既可以作为通用的I/O口进行数据的输入/输出，也可以作为单片机系统的地址/数据线使用。

如果端口的负载恰是一个晶体管的基极，导通了的PN结会把端口引脚的高电平拉低，这样直接读引脚就会把本来的“1”误读为“0”。 但若从锁存器Q端读，就能避免这样的错误，得到正确的数据。

P1端口

与P0口不同处：首先它不再需要多路转接电路MUX；其次是电路的内部有上拉电阻，与场效应管共同组成输出驱动电路。

电阻接电源=上拉电阻

电阻接地=下拉电阻

P2端口

通常情况下，P2口是作为高位地址线使用而不作为数据线使用

芯片引脚及功能

供电引脚

供电引脚接入AT89S51的工作电源

  • VCC:电源正极，一般为+5V。
  • GND：电源接地。

I/O引脚

p0

当访问外部程序和数据存储器时，P0口也被作为低8位地址/数据复用。在这种模式下，P0具有内部上拉电阻。

在程序校验时，输出指令字节。程序校验时，需要外部上拉电阻。

控制引脚

  • RST：复位输入
  • ALE/PROG：地址锁存控制信号是访问外部程序存储器时，锁存低8位地址的输出脉冲。
  • PSEN：外部程序存储器选通信号
  • EA/VPP:访问外部程序存储器控制信号

外接晶振引脚（时钟信号）

  • XTAL1：接外部晶体的一个引脚，在单片机内部，它是一个反相放大器的输入端。
  • XTAL2: 接外部晶体的另一端，在单片机内部接到反相放大器的输出端，当采用外接晶体振荡器时，此引脚可以不接。

单片机工作时序

时序定时单位

拍节与状态

把振荡脉冲的周期定义为拍节（用P表示）。振荡脉冲经过二分频后，就是单片机的时钟信号的周期，其定义为状态（用S表示）。其前半周期对应的拍节叫拍节1，后半周期对应的拍节叫拍节2.

机器周期（一个机器周期=12个振荡周期）

一个机器周期的宽度为6个状态，并依次表示为S1~S6.

一个状态包括两个拍节，因此一个机器周期总共有12个拍节，分别记作S1P1、S1P2、…、S6P2.当振荡脉冲频率为12MHZ时，一个机器周期为1us，当振荡脉冲频率为6MHz时，一个机器周期为2us。

指令周期

指令周期是最大的时序定时单位，执行一条指令所需要的时间称为指令周期。

一个机器周期的指令称为单周期指令，包含两个机器周期的指令称为双周期指令。

MCS-51 单片机通常可以分为单周期指令、双周期指令和四周期指令3种。四周期指令只有乘法和除法两条指令，其余均为单周期和双周期指令。

单片机的指令系统

指令格式

操作码[目的操作数]，[源操作数];[注释]

符号说明

  • #data: 8位立即数
  • #data16: 16位立即数
  • Rn：工作寄存器，R0~R7，n为0~7
  • Ri：工作寄存器，0或1，i=0或1.
  • @Ri:寄存器Ri间接寻址8位存储单元00H~FFH
  • direct：8位直接寻址，可以是特殊功能寄存器（SFR）的80H~FFH或内部存储单元00H~7FH
  • addr16: 16位目的地址，用于LJMP和LCALL指令，可在64KB地址内转移或调用。
  • rel：带符号的8位偏移地址，主要应用于所有的条件转移指令和SJMP。其范围是相对于下一条指令的第一字节地址-128~+127字节。
  • bit：位地址。片内RAM中的可寻址位和专用寄存器中的可寻址位
  • DPTR：数据指针，可用于16位的地址寄存器
  • @:间接寄存器或基址寄存器的前缀，如@DPTR、@Ri、@A+PC、@A+DPTR.
  • A:累加器ACC
  • B：通用寄存器，常用于乘法MUL和除法
  • CY：进位标志位或者布尔处理器中的累加器
  • /:位操作数前缀，表示对该位操作数进行取反操作。
  • (x)：寄存器或存储单元x的内容
  • （（x））：以寄存器或存储单元x的内容作为地址的存储单元的内容

单片机的寻址方式

立即寻址

直接给出操作数的地址。直接将需要访问的数据在指令中给出，这样的寻址方式就是立即寻址。

MOV A，#dataH

这是一条立即寻址方式的传送指令，通常出现在指令中的操作数data称为立即数。

注意：在立即数寻址中立即数前面必须加上一个#号

直接寻址

直接寻址就是直接在指令中指定操作数的地址
