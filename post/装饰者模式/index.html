装饰者模式

Oct 25, 2018 2 min read java, 设计模式

装饰者模式

装饰者模式的定义

装饰者模式动态的将责任附加到对象上。若要扩展功能，装饰者提供了比继承更有贪心的替代方案

装饰者模式的结构

Decorator–是装饰者的父类，每个装饰者都需要继承这个抽象类(或实现这个接口)

ConcreteDecoratorA/B–具体的装饰者，就对应上述例子的加冰，加糖等.

ConcreteComponent–具体的对象，就是上述例子中的可乐.

Component–装饰者模式中的最顶级的父类，装饰者与被装饰者都是它的子类或实现类才行。

例子

类图:

最顶级的父类:

public abstract class Beverage {
    String description="Unknown Beverage";

    public String getDescription() {
        return description;
    }
    public abstract double cost();
}

被装饰的Espresso咖啡:

package com.company;

/**
 * Espresso扩展自Beverage类，因为Espresso是一种饮料
 */
public class Espresso extends Beverage {
    @Override
    public double cost() {
        return 1.99;
    }//计算Espresso的价钱，不用管调料的价钱

    public Espresso(){
        description="Espresso";
    }
}

被装饰的DarkRoast咖啡:

public class DarkRoast extends Beverage {
    @Override
    public double cost() {
        return 0.99;
    }

    public DarkRoast(){
        description="DarkRoast";
    }
}

被装饰的HouseBlend咖啡:

public class HouseBlend extends Beverage {
    @Override
    public double cost() {
        return 0.89;
    }

    public  HouseBlend(){
        description="HouseBlend";
    }
}

Condiment类，所有装饰类的父类

public abstract class Condiment extends Beverage{
    public abstract String getDescription();//所有的调料装饰者都必须重新实现getDescription()方法。
}

装饰类:Mocha类

package com.company;

/**
 * 摩卡作为一个装饰者，所以让它扩展自Condiment
 */
public class Mocha extends Condiment {
    Beverage beverage;//用一个实例变量记录饮料，也就是被装饰者

    public  Mocha(Beverage beverage){
        this.beverage=beverage;
    }
    //想办法让被装饰者被记录到实例变量中。这里的做法是：把饮料当做构造器参数，再由构造器将此饮料记录在实例变量中。

    @Override
    public String getDescription() {
        return beverage.getDescription()+", Mocha";
    }

    @Override
    public double cost() {
        return 0.20+beverage.cost();
    }

}

装饰类：Soy类

public class Soy extends Condiment {
    Beverage beverage;
    public Soy(Beverage beverage){
        this.beverage=beverage;
    }

    @Override
    public String getDescription() {
        return beverage.getDescription()+", Soy";
    }

    @Override
    public double cost() {
        return beverage.cost()+0.15;
    }
}

装饰类：whip类

public class Whip extends Condiment {
    Beverage beverage;
    public Whip(Beverage beverage){
        this.beverage=beverage;
    }

    @Override
    public String getDescription() {
        return beverage.getDescription()+", Whip";
    }

    @Override
    public double cost() {
        return beverage.cost()+0.10;
    }
}

以及测试类:

public class Main {

    Beverage beverage=new Espresso();//订一杯Espresso,不需要调料

    Beverage beverage2=new DarkRoast();//制造一个DarkRoast对象
